SITE UPLOADEN

Uploaden

Nu je een website hebt gemaakt wil je natuurlijk dat andere mensen deze website ook kunnen bezoeken. Als je een website maakt en zelf bezoek via localhost, of door de .html file te openen kan je zelf de website zien. Maar andere mensen kunnen hem nog niet zien. Daarom is het van belang dat je de website upload naar een host of webserver. In dit geval maken we gebruik van een Amazon S3 container die de website host. Hierdoor komt de website openbaar te staan voor iedereen, en kan iedereen ook de website bezoeken. Voor het uploaden heb ik gebruik gemaakt van Cloudberry. Een programma om toegang te krijgen tot het filesystem van de S3 container van de CHE FDD server. Je logt dan in met de inloggegevens die je al hebt, of hebt gekregen en kan dan de bestanden van de website uploaden. Hierdoor komen ze op de server te staan en zijn ze zichtbaar voor iedereen die de website wilt bezoeken.

Het gebruik van Cloudberry Explorer is eigen vrij eenvoudig. Je klikt bij Source op New Source. Hier kies je de Amazon S3 optie, aangezien dat is wat gebruikt wordt. Vervolgens krijg je een scherm waar je de access en secret key moet invoeren. Deze hebben we gekregen via de mail. En verolgens kan je al verbinding maken. Je moet nu alleen nog de locatie van de folder kiezen, door de standaard root locatie aan te passen naar fdd.hbo-ict.resources/2223/tmellink in mijn geval. En dan krijg je de juiste folder te zien waar je de website files kan uploaden.

DATACENTERS

Datacenters

Bijna alles wat online zichtbaar is staat opgeslagen op een datacenter. Dit zijn hele grote servers die allemaal verbonden zijn met het internet. Hierdoor zijn websites ook beschikbaar over de hele wereld. Datacenters staan overal over de hele wereld, en hierdoor kan het bijvoorbeeld dat de website van Bol.com niet in Nederland gehost is, maar in Duitsland.

DNS

Dns

Het tonen van een website is echter niet zo eenvoudig als vele mensen denken. Aan de buitenkant lijkt het heel simpel, je typt de URL / domein van een website in en voila je site de website. Maar zo simpel werkt het achter de schermen niet helemaal. Elk apparaat dat verbonden is met het internet heeft een eigen IP-adres. Dit is een unieke volgorde van nummers die jou een unieke identiteit geven. Maar ook Datacenters / hosting servers hebben een eigen IP-adres. Zodat deze ook unieke identiteit hebben. Het domein van een website staat verbonden aan het IP-adres van de host van de website. Een domein zelf zou eigenlijk geen verbinding kunnen maken met de website. Hiervoor word gebruik gemaakt van DNS servers. Dit zijn servers die een domeinnaam vertalen naar het verbonden IP-adres. Zodat jij de website te zien krijgt. Als je dus naar Google.com gaat wordt er eerst een verzoek gestuurd naar een DNS server die dan een antwoord terug stuurt met het IP-adres van Google.com zodat je ook daadwerkelijk verbinding kan maken met de servers van Google en de website te zien krijgt. Dit is heel simpel uitgelegd hoe het internet werkt.
